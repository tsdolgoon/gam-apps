# GAM Back Office — Голомт Ассет Менежмент

Бэк-офис систем: хөрөнгө оруулагчийн бүртгэл, нэгж эрхийн **худалдан авалт / буцаан худалдалт**,
сангийн мэдээлэл, нэгж эрхийн үнэ (НЦХҮ). Бүх өгөгдөл таны **OneDrive** дээрх нэг Excel файлд хадгалагдана.
Сервер, мэдээллийн сан, IT шаардлагагүй.

> Back-office app for Golomt Asset Management: investor registry, fund unit **subscription / redemption**,
> fund master data, and unit price (NAV). All data lives in a single Excel workbook on your OneDrive.
> No server, no database, no IT required.

---

## 1. Шаардлага · Requirements
- **Microsoft Edge** эсвэл **Google Chrome** (File System Access API шаардлагатай).
- **OneDrive** desktop клиент компьютерт суусан, нэвтэрсэн байх (Microsoft 365).

## 2. Эхлүүлэх · How to start
1. `GAM Backoffice эхлүүлэх.cmd` файл дээр **давхар товшино** (Edge дээр нээгдэнэ).
   *(эсвэл `index.html`-ийг Edge/Chrome-оор нээнэ.)*
2. Эхний удаа **«Шинээр үүсгэх»** дарж, өгөгдлийн файлаа **OneDrive доторх хавтаст**
   `GAM_BackOffice.xlsx` нэрээр хадгална. Хоёр сан (Тогтмол Орлого, Урбан Скай) автоматаар ачаалагдана.
3. Дараа дараагийн удаад апп нь сүүлийн файлыг **санаж**, шууд холбогдоно
   (эхний удаа нэг товшилтоор зөвшөөрөл өгнө).

> First run → **Create new** → save `GAM_BackOffice.xlsx` **inside a OneDrive folder**.
> The app remembers the file next time (one click to re-grant access).

## 3. Хэрхэн ажилладаг вэ · How it works
- Та өгөгдөл оруулах бүрт апп файлыг **автоматаар хадгална**, OneDrive үүлэн рүү синк хийнэ.
- Дээд талын **«Хадгалах»** товчоор гараар хадгалж, **↻** товчоор файлаас дахин ачаална.
- Өгөгдлийг хүссэн үедээ Excel-ээр нээж харж/хэвлэж болно (sheets: Funds, Investors, Transactions, NAVHistory).

## 4. Модулиуд · Modules
| Цэс | Зориулалт |
|---|---|
| **Хяналтын самбар** | Нийт AUM, сан тус бүрийн төлөв, сүүлийн гүйлгээ |
| **Хөрөнгө оруулагч** | Бүртгэл (KYC), эзэмшил, гүйлгээний түүх. **«Тайлан»** товчоор A4 хэвлэхэд бэлэн хөрөнгө оруулагчийн тайлан (сар/улирал/жил) үүсгэж, хөтчийн хэвлэх цонхоор хэвлэнэ — сан тус бүрийн дэлгэрэнгүй, гүйлгээний түүх, гүйцэтгэлийн нэгдсэн дүн. |
| **Худалдан авалт** | Дүн оруулахад нэгж эрх автоматаар тооцоологдоно (дүн ÷ нэгж эрхийн үнэ) |
| **Буцаан худалдалт** | Нэгж эрх буцаахад дүн тооцоологдоно; эзэмшлээс хэтрэхээс хамгаална |
| **Гүйлгээний түүх** | Шүүлт, цуцлах, CSV татах |
| **Сангууд** | Сангийн үндсэн мэдээлэл (засварлах боломжтой) |
| **НЦХҮ / NAV** | Нэгж эрхийн үнэ = Цэвэр хөрөнгө ÷ Гүйлгээнд буй нэгж |
| **Бондын багц · Bond Portfolio** | Сан тус бүрийн бондын бүртгэл. Зах зээлийн үнэ (par-ийн %) гараар шинэчилнэ; YTM, хуримтлагдсан хүү, зах зээлийн үнэлгээ, боломжит ашиг/алдагдал автоматаар тооцоологдоно. Сан тус бүрийн дүн, мөнгөн урсгалын хуваарь харагдана. |
| **Хөрөнгө оруулалтын хэрэгсэл · Investment Instruments** | Бүх санхүүгийн хэрэгслийн нэгдсэн бүртгэл (Бонд / Хувьцаа / Сангийн нэгж / Мөнгөний ЗЗ / Бусад). Төрөл сонгоход тухайн төрлийн талбарууд гарч ирнэ. Зах зээлийн үнэлгээ, ашиг/алдагдал %, сангийн жин (%) — бондын YTM, хуримтлагдсан хүү автомат тооцоо. Төрөл/сан/төлвөөр шүүх, нэр/гаргагчаар хайх, үнийн түүх гараар бүртгэх. |
| **Шимтгэл ба татвар** | Удирдлагын төлбөр (худалдан авах үед жилийн % урьдчилан суутгана, тусдаа мөрөөр), Гүйцэтгэлийн төлбөр (1 жилийн дараа, босго өгөөжөөс давсан өгөөжид) ба өгөөжийн татвар (Дотоодын 10% / Гадаадын 20%). «Calculate Fees» товчоор 1 жил болсон бүх хөрөнгө оруулалтын гүйцэтгэлийн шимтгэл/татварыг автоматаар тооцно. Хөрөнгө оруулагч тус бүрийн задаргаа ба нэгдсэн дүн харагдана. |
| **ECIF — Хуримтлал** | Ажилчдын хуримтлалын сан. Оролцогч байгууллага бүр **Тогтмол Орлого санд нэг нэрлэсэн данс эзэмшигч** болно; ажилтнууд дэд данстай. Хувь нэмэр → SGF нэгж эрх → ажилтанд хуваарилагдана. Татан авалт боломжтой. |

## 5. Өгөгдлийн бүтэц · Data model (Excel sheets)
- **Funds** — сангийн мастер мэдээлэл (удирдлага/гүйцэтгэлийн шимтгэл %, босго өгөөж % орно)
- **Investors** — хөрөнгө оруулагчийн бүртгэл (харьяалал: Дотоодын/Гадаадын — татварын хувь тодорхойлно)
- **Transactions** — худалдан авалт / буцаан худалдалт, мөн удирдлагын/гүйцэтгэлийн шимтгэл, татварын мөрүүд
- **NAVHistory** — өдрийн НЦХҮ ба нэгж эрхийн үнэ
- **Bonds** — бондын багц (гаргагч, төрөл, купон, хугацаа, үнэ); YTM, хуримтлагдсан хүү, зах зээлийн үнэлгээ, ашиг/алдагдал автоматаар тооцоологдож хадгалагдана
- **Instruments** — бүх хэрэгслийн нэгдсэн бүртгэл (төрөл, нэр, гаргагч, ISIN, сан, үнэ, тоо); зах зээлийн үнэлгээ, ашиг/алдагдал %, YTM, хуримтлагдсан хүү автоматаар тооцоологдож хадгалагдана
- **PriceHistory** — хэрэгслийн үнийн түүх (гараар бүртгэсэн)
- **Fees** — удирдлага/гүйцэтгэлийн шимтгэлийн тооцоо
- **ECIFFirms** — ECIF-д оролцогч байгууллага (тус бүр SGF-д нэрлэсэн данстай)
- **ECIFEmployees** — оролцогч байгууллагуудын ажилтнууд (дэд данс)
- **ECIFContributions** — ажилтны хувь нэмэр / татан авалтын дэвтэр
- **Meta** — системийн мэдээлэл

### Техник тал · Tech notes
- Vanilla HTML/CSS/JS + [SheetJS](https://sheetjs.com) (offline, `assets/xlsx.full.min.js`).
- Файлд хандалт: **File System Access API**; файл сонголтыг IndexedDB-д сануулна.
- `tools/serve.ps1` — зөвхөн хөгжүүлэлт/туршилтад зориулсан локал статик сервер (заавал биш).
